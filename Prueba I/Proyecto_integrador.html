1. La Naturaleza del Problema

🐌

Intensivo en Cómputo (CPU-Bound)

Generar hashes (MD5, SHA-1) requiere millones de operaciones criptográficas. Un enfoque secuencial (1 núcleo) escala linealmente, volviéndose inviable frente a diccionarios enormes.

🚀

"Embarrassingly Parallel"

Cada contraseña se evalúa de forma 100% independiente. Cero dependencias de datos. Es el escenario ideal para distribuir carga en arquitecturas masivas (Multinúcleo y GPUs).

2. El Dataset: RockYou

🗄️
rockyou.txt

El Estándar de Benchmarking

  • ➤
    Volumen: Más de 14.3 Millones de contraseñas reales.
  • ➤
    Formato: Texto plano.
  • ➤
    Complejidad: Alta varianza en longitudes y caracteres, ideal para probar divergencia en GPUs.

⚠️ NOTA ÉTICA: El uso de este dataset en el curso es exclusivamente académico para medición de rendimiento (HPC), no para vulnerar sistemas de terceros.

3. Paradigmas a Evaluar

🐢

Secuencial

Línea Base (C++)

1 núcleo. Fundamental para obtener el tiempo inicial ($T_1$) y calcular el Speedup del resto.

🧩

OpenMP

Multinúcleo CPU

Memoria compartida. Aprovecha todos los hilos físicos del procesador particionando el diccionario.

🟩

CUDA

GPU NVIDIA

Paralelismo masivo SIMT. Cada thread de la gráfica procesa un hash. Altísimo throughput esperado.

🟥

OpenCL

GPU AMD / Híbrido

Computación heterogénea y código abierto. Mide portabilidad vs rendimiento propietario.

4. Métricas y Riesgos

Aceleración Teórica (Speedup)

Multiplicador de rendimiento respecto a 1 núcleo.

Riesgo: Warp Divergence

Variabilidad del largo de contraseñas destruye eficiencia en GPU.

🏁

Conclusión Esperada

Demostrar empíricamente que, para tareas de Ciencia de Datos intensivas en cómputo y sin dependencias (Embarrassingly Parallel), el procesamiento en GPU (CUDA/OpenCL) justifica ampliamente la complejidad de desarrollo, reduciendo radicalmente los tiempos frente al multinúcleo de CPU.
